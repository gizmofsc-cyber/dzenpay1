# Деплой на Vercel - Пошаговая инструкция

## 1. Подготовка проекта

✅ Проект уже готов к деплою - все ошибки сборки исправлены.

## 2. Настройка базы данных на Vercel

### Вариант A: PostgreSQL (Рекомендуется)

1. В панели управления Vercel перейдите в раздел "Storage"
2. Создайте новую PostgreSQL базу данных
3. Скопируйте строку подключения (DATABASE_URL)

### Вариант B: SQLite (Временное решение)

1. Используйте встроенную SQLite базу данных
2. DATABASE_URL будет автоматически настроен

## 3. Настройка переменных окружения

В панели управления Vercel добавьте следующие переменные:

```
DATABASE_URL=postgresql://username:password@host:port/database
NEXTAUTH_URL=https://your-domain.vercel.app
NEXTAUTH_SECRET=your-secret-key-here
NODE_ENV=production
```

## 4. Деплой проекта

1. Подключите GitHub репозиторий к Vercel
2. Vercel автоматически выполнит сборку
3. Дождитесь завершения деплоя

## 5. Инициализация базы данных

После успешного деплоя выполните инициализацию базы данных:

### Через Vercel CLI:
```bash
vercel env pull
npm run vercel:init
```

### Через API endpoint:
После деплоя откройте: `https://your-domain.vercel.app/api/test-db`

## 6. Проверка работы

### Тестовые endpoints:
- `https://your-domain.vercel.app/api/test-env` - проверка переменных окружения

### Основные страницы:

## 8. Возможные проблемы и решения

### Ошибка 404:
- Проверьте правильность URL
- Убедитесь, что проект успешно задеплоен

### Ошибка 500:
- Проверьте переменные окружения
- Убедитесь, что база данных доступна
- Проверьте логи в панели управления Vercel

### Ошибки аутентификации:
- Проверьте NEXTAUTH_SECRET
- Убедитесь, что NEXTAUTH_URL соответствует домену

## 9. Мониторинг

- Используйте панель управления Vercel для мониторинга
- Проверяйте логи в разделе "Functions"
- Следите за использованием базы данных

## 10. Обновления

Для обновления проекта:
1. Внесите изменения в код
2. Сделайте коммит и push в GitHub
3. Vercel автоматически пересоберет и задеплоит проект
